# Workstreams

This file breaks the roadmap into practical engineering tracks that can run in parallel, with current priority given to CTF workflows and shell ergonomics.

## 1. CTF Command Model

Primary outcome: the CTF workflow is coherent, learnable, and fast under pressure.

### Near-term tasks

- Define the preferred lifecycle from event creation to challenge archival.
- Audit overlap and ambiguity between `add`, `work`, `solve`, `done`, `finish`, and `archive`.
- Normalize naming, help text, and examples around the main CTF path.
- Decide what must stay human-friendly versus machine-friendly in command output.

### Success measure

- A user can understand the intended CTF flow without guessing which command to use next.

## 2. Shell Integration And Completion

Primary outcome: Wardex feels native in Bash and Zsh.

### Near-term tasks

- Design completion support for subcommands, event names, categories, challenge names, and likely flags.
- Decide whether completions are generated by Clap, maintained as scripts, or exposed through a dedicated command.
- Add shell install instructions and a minimal setup path for Bash and Zsh.
- Review `ctf path --cd`, `ctf work`, and similar commands for eval-safe shell usage.

### Success measure

- Tab completion and shell navigation meaningfully reduce typing and context switching.

## 3. Quality And Safety

Primary outcome: competition-critical commands are trustworthy.

### Near-term tasks

- Add targeted integration tests for lifecycle flows, archive imports, undo-adjacent operations, and malformed metadata.
- Review guardrails for destructive operations and accidental path selection.
- Add regression tests for context resolution, fuzzy matching, and shell-oriented output.
- Remove or resolve current compiler warnings in the test suite.

### Success measure

- High-risk CTF commands have regression coverage and clear failure behavior.

## 4. Templates And Writeups

Primary outcome: category-specific workflows feel intentional instead of generic.

### Near-term tasks

- Improve challenge scaffolding and category-specific templates.
- Define a default notes structure that feeds writeup generation cleanly.
- Review how `solve`, `done`, and `writeup` interact with notes, flags, and archives.
- Add example workflows for solo events and quick challenge pivots.

### Success measure

- The default challenge folder feels useful immediately after creation or import.

## 5. Distribution And Developer Operations

Primary outcome: releases and shell features are easy to install and verify.

### Near-term tasks

- Add a simple CI workflow with formatting, linting, and tests.
- Add completion verification steps to release readiness.
- Validate Nix and Cargo installation paths against shell integration docs.
- Document platform assumptions and known limitations for Bash and Zsh users.

### Success measure

- Releases are repeatable and documented without relying on tribal knowledge.

## Suggested First Backlog

1. Define the canonical CTF lifecycle and simplify command guidance around it.
2. Design and implement Bash/Zsh completion support for core CTF flows.
3. Expand integration tests around `ctf path`, `ctf work`, `ctf import`, `ctf solve`, and `ctf finish`.
4. Review output formatting and help text for shell-safe usage.
5. Improve challenge templates and writeup conventions after the command surface is stable.
